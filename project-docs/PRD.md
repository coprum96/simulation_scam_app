# Product Requirements Document

## Название

Симулятор уязвимости банковского приложения и цифрового кошелька

## Problem Statement

Пользователь мобильного банка или цифрового кошелька может совершать рискованные действия под давлением, в состоянии тревоги, по инструкции мошенника или из-за непонятного интерфейса. В таких сценариях важно изучать поведение пользователя, эффективность предупреждений и сигналы повышенного риска.

## Product Goal

Создать русскоязычную исследовательскую платформу, в которой можно моделировать безопасные и опасные пользовательские сценарии, собирать поведенческие события, рассчитывать score риска и анализировать уязвимость пользователя к социальной инженерии.

## Целевая аудитория

1. Исследователь лаборатории
2. Anti-fraud analyst
3. Product manager / UX researcher
4. Преподаватель / студент
5. Партнёр / стейкхолдер, которому показывают demo

## Основные user stories

- Как исследователь, я хочу запускать разные сценарии мошенничества и смотреть, как меняется поведение пользователя.
- Как аналитик, я хочу видеть таймлайн действий и список risk flags.
- Как продакт-менеджер, я хочу тестировать разные warning screens и friction patterns.
- Как преподаватель, я хочу демонстрировать студентам реальные механики scam-сценариев без риска для пользователей.
- Как исследователь кошельков, я хочу моделировать опасные подтверждения и blind-signing сценарии.

## MVP Scope

### Входит
- Scenario Hub
- Banking Simulator
- Wallet Simulator
- Telemetry Logger
- Rule-based Risk Engine
- Session Replay
- Research Dashboard
- Export JSON

### Не входит
- Backend
- Реальные интеграции
- Настоящие переводы
- Реальная регистрация/логин
- ML-модель в production
- Поддержка нескольких языков

## Основные продуктовые модули

### 1. Scenario Hub
Карточки сценариев, фильтры, описание риска, запуск сценария.

### 2. Banking Simulator
Экраны:
- Главный экран
- Счета
- Перевод
- Добавление получателя
- Проверка данных
- Предупреждение
- Подтверждение
- Результат

### 3. Wallet Simulator
Экраны:
- Главный экран кошелька
- Активы
- Подключение сервиса
- Подпись операции
- Разрешение на доступ к активам
- Экран восстановления
- Итог

### 4. Telemetry Layer
Сбор и хранение событий пользователя в рамках сессии.

### 5. Risk Engine
Правила, score, флаги, объяснения.

### 6. Session Review
Таймлайн событий, итоговый риск, пояснения.

### 7. Dashboard
Список сессий, сравнение, статистика по сценариям и профилям.

## KPI MVP

- Можно запустить минимум 6 сценариев.
- Для каждой сессии сохраняются события.
- Для каждой сессии рассчитывается score риска.
- Дашборд показывает сессии и флаги риска.
- Все тексты интерфейса естественно звучат на русском.
- Приложение корректно работает на desktop и mobile.

## Ограничения

- Только mock data
- Только frontend
- Никакого real banking branding
- Кодовые идентификаторы могут быть на английском, UI только на русском
